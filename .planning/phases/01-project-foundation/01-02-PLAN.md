  <action>
    1. Install Supabase dependency (if not already from Plan 01):
       ```
       cd apps/mobile && npx expo install @supabase/supabase-js expo-sqlite
       ```

    2. Create `apps/mobile/src/lib/supabase.ts` per RESEARCH.md Pattern 1:
       - Import `'expo-sqlite/localStorage/install'` at the very top (polyfill)
       - Import `createClient` from `@supabase/supabase-js`
       - Read env vars with `EXPO_PUBLIC_` prefix: `process.env.EXPO_PUBLIC_SUPABASE_URL!` and `process.env.EXPO_PUBLIC_SUPABASE_ANON_KEY!`
       - Create and export `supabase` client with auth config:
         - `storage: localStorage` (expo-sqlite polyfill)
         - `autoRefreshToken: true`
         - `persistSession: true`
         - `detectSessionInUrl: true` (CRITICAL for web magic link PKCE flow -- do NOT set to false)
       - Do NOT use `@react-native-async-storage/async-storage` (deprecated for Expo)

    3. Create `apps/mobile/src/contexts/auth.tsx`:
       - Create `AuthContext` with React.createContext
       - Export `AuthProvider` component that:
         - Holds `session` state (Session | null) and `isLoading` state (boolean, starts true)
         - On mount, calls `supabase.auth.getSession()` to get initial session
         - Subscribes to `supabase.auth.onAuthStateChange()` to update session reactively
         - Cleans up the auth listener on unmount
         - Provides `session`, `isLoading`, `signIn(email: string)`, `signOut()` via context
       - `signIn` calls `supabase.auth.signInWithOtp({ email })` -- returns the result for the caller to handle success/error UI
       - `signOut` calls `supabase.auth.signOut()`
       - Export `useSession()` hook that reads from AuthContext (throw if used outside provider)
       - Type the context properly with TypeScript (no `any`)

    4. Update `apps/mobile/app/_layout.tsx`:
       - Import and wrap the entire navigation tree with `<AuthProvider>`
       - Replace the simple Stack.Screen entries with `Stack.Protected` guards per RESEARCH.md Pattern 3:
         - `<Stack.Protected guard={!!session}>` around `<Stack.Screen name="(dashboard)" />`
         - `<Stack.Protected guard={!session}>` around `<Stack.Screen name="(auth)" />`
       - Use `useSession()` inside a child component (e.g., `RootNavigator`) to access session, since the hook must be called inside `AuthProvider`
       - While `isLoading` is true, return null (splash screen stays visible)
       - Keep the existing font loading logic -- both fonts and auth must be ready before rendering

    5. Update `apps/mobile/app/(auth)/login.tsx`:
       - Add a TextInput for email (styled: dark surface bg, white text, white/15 border, Inter font)
       - Wire the gradient "SEND MAGIC LINK" button to call `signIn(email)` from `useSession()`
       - Show loading state on the button while the request is in flight (e.g., "SENDING..." text, disabled state)
       - On success: show a message "Check your email for the magic link!" in green/accent text
       - On error: show the error message in danger text below the button
       - Keep the "FUELED.SHOWCASE" branding from Plan 01

    6. Update `apps/mobile/app/(dashboard)/(tabs)/profile.tsx`:
       - Use `useSession()` to get the current session
       - Display `session.user.email` in the "Signed in as" card (replace placeholder text)
       - Wire the "SIGN OUT" button to call `signOut()` from `useSession()`

    7. Create `apps/mobile/.env.example`:
       ```
       EXPO_PUBLIC_SUPABASE_URL=your-project-url
       EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
       ```

    8. Create `apps/mobile/.env.local` with placeholder values (user will fill in real values):
       ```
       EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
       EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
       ```
       Note: The executor should prompt the user for real Supabase credentials. The `.env.local` file is gitignored.

    9. Set up Supabase CLI project structure:
       - Create `infra/supabase/` directory
       - Create `infra/supabase/package.json` with `{ "name": "supabase", "private": true }` (workspace member)
       - If `supabase` CLI is available: run `npx supabase init` in `infra/supabase/` to generate `config.toml`
       - If not available: create a minimal `config.toml` with `[api]` and `[auth]` sections enabling email auth and magic link
       - Create `infra/supabase/.env.example` with Supabase project secrets template
       - Create `infra/supabase/.env.local` placeholder (gitignored)

    10. Verify the auth flow compiles:
        - Run `npx expo export --platform web` to confirm no build errors
        - Check that TypeScript has no errors in the auth context and supabase client
