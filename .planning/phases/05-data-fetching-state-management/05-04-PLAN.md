  <behavior>
    - useCacheEntries() returns an array of cache entry objects { queryKey, status, dataUpdatedAt, data, dataLength }
    - useCacheEntries subscribes to QueryCache changes and updates on every cache event
    - State management screen renders each cache entry as a styled card
    - Each card shows: queryKey as formatted label, status badge, data preview as JSON, last updated time
    - Screen updates in real time as cache changes
  </behavior>
  <action>
    1. Create `src/hooks/use-cache-entries.ts` per RESEARCH.md Pattern 7:
       - Import useQueryClient from @tanstack/react-query
       - Use useState for entries array, useEffect to subscribe to cache changes
       - On each cache event: getAll() from query cache, map each query to:
         ```
         {
           queryKey: query.queryKey,
           status: query.state.status,
           dataUpdatedAt: query.state.dataUpdatedAt,
           data: query.state.data,
           dataLength: Array.isArray(query.state.data?.items) ? query.state.data.items.length : null,
         }
         ```
       - Return the entries array
       - Clean up subscription on unmount

    2. Replace placeholder `app/(dashboard)/state-management.tsx` with the cache viewer screen:
       - Title section: "State Management" (Typography h2) + subtitle "Live TanStack Query cache viewer -- read-only snapshot of current app state"
       - ScrollView with max-w-3xl mx-auto layout (consistent with other screens)
       - Stats bar at top: total cache entries count, entries by status (success/pending/error)
       - Each cache entry rendered as a Card with:
         - Query key formatted as human-readable label (e.g., `items > page:1 search:"" category:""`)
         - Status as Badge (success=green, pending=info, error=error)
         - Last updated: relative time (e.g., "2 min ago") or "never" if dataUpdatedAt is 0
         - Data preview: formatted JSON inside a styled code block (bg-black/50, border border-white/10, rounded-xl, monospace font, max-height with scroll)
         - JSON.stringify(entry.data, null, 2) for formatting, truncated to first 500 chars with "..." if longer
       - Empty state: "No cache entries yet. Navigate to Data Fetching to populate the cache."
       - This screen is READ-ONLY per user decision -- no buttons, no demo triggers, just the viewer
       - ScrollToTop FAB (reuse existing component)

    3. Write `tests/screens/state-management.test.tsx`:
       - Mock `@tanstack/react-query`'s useQueryClient to return a mock queryClient with getQueryCache().getAll() returning mock entries and subscribe() returning an unsubscribe fn
       - Test: renders "State Management" heading
       - Test: renders cache entries when cache has data
       - Test: renders empty state when cache is empty
       - Test: displays query key and status for each entry
