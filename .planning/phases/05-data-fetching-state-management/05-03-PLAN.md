  <behavior>
    - useItems({ page, search, category, priority, status }) returns { data: { items, totalCount, totalPages, currentPage }, isPending, isError, error }
    - useItems uses queryKey ['items', { page, search, category, priority, status }] so filter changes trigger refetch
    - useItems calls supabase.from('items').select('*', { count: 'exact' }).order('created_at', { ascending: false }).range(from, to) with optional .ilike('title', '%search%'), .eq('category', cat), .eq('priority', pri), .eq('status', stat)
    - PAGE_SIZE is 10
    - useCreateItem calls supabase.from('items').insert(input).select().single() and invalidates ['items'] on settled
    - useUpdateItem calls supabase.from('items').update(input).eq('id', id).select().single() with optimistic cache update
    - useDeleteItem calls supabase.from('items').delete().eq('id', id) with optimistic removal, snapshot rollback on error, invalidation on settled
    - useDebouncedValue(value, delay=300) returns debounced value after delay ms
  </behavior>

    2. Create `src/hooks/use-items.ts` with four hooks:
       - **useItems**: Per RESEARCH.md Pattern 3. Accepts `{ page, search, category, priority, status }`. Returns paginated data with totalCount/totalPages. Applies filters only when non-empty. PAGE_SIZE = 10.
       - **useCreateItem**: useMutation that inserts via Supabase (chain `.select().single()` per Pitfall 2), invalidates ['items'] onSettled.
       - **useUpdateItem**: useMutation with optimistic update. onMutate: cancel queries, snapshot, setQueriesData to update item in-place. onError: rollback. onSettled: invalidate.
       - **useDeleteItem**: Per RESEARCH.md Pattern 4. useMutation with optimistic removal. onMutate: cancel queries, snapshot, remove item from all cached pages. onError: rollback from snapshot. onSettled: invalidate ['items'].
