  <action>
    Add count-up animation to the stat numbers in Meet Jim. Per RESEARCH.md, use a JS-based approach (not Reanimated createAnimatedComponent -- unreliable on web for text props).

    1. Create a `CountUp` component inline in meet-jim.tsx (or as a small helper):
       - Props: `to: number` (target value), `duration?: number` (default 1500ms)
       - Use `useState` for displayed value (starts at 0)
       - Use `useEffect` on mount: start a `requestAnimationFrame` loop that increments from 0 to `to` over `duration` ms using easeOutCubic easing
       - On each frame: calculate elapsed time, compute progress (0-1) with easing, set displayed value to `Math.round(progress * to)`
       - Clean up animation frame on unmount
       - Render the number via Typography

    2. Replace the static `{stat.value}` in the Stats Grid with `<CountUp to={stat.value} />` for each stat.

    3. Ensure the animation only runs once on mount -- no re-triggers on scroll or state changes.

Or, since the test environment won't run requestAnimationFrame, mock it so the final value renders immediately. The simplest approach: in the test, use `jest.useFakeTimers()` and advance timers, or just assert the stat labels are present (the values will be 0 initially in test since rAF doesn't run in jsdom -- that's acceptable).

<task type="auto">
  <name>Task 2: Fix accordion content truncation and tech stack subtitle wrapping</name>

  <action>
    **Accordion fix** -- The bug is in AccordionItem: `maxHeight: height.value * 500` caps expanded height at 500px, which truncates long content (e.g., Component Library has 17 items).

    1. Fix `src/components/layout/Accordion.tsx`:
       - Add state for measured content height: `const [contentHeight, setContentHeight] = useState(0)`
       - Wrap the content `<View>` inside AccordionItem body with `onLayout={(e) => setContentHeight(e.nativeEvent.layout.height)}`
       - Change the animated style from `maxHeight: height.value * 500` to `maxHeight: height.value * (contentHeight || 2000)`
       - This way: when content is measured, it animates to exact height; before measurement, it uses 2000 as a safe fallback
       - Keep overflow: 'hidden' in the animated style (needed for collapse animation)
       - The onLayout View should NOT have overflow hidden -- it needs to render at full height to be measured

    2. **Subtitle wrapping fix** in `app/(dashboard)/the-showcase.tsx`:
       - Find the tech stack card section where subtitle/rationale text is rendered
       - The issue is the card container uses `style={{ minWidth: '46%', flexGrow: 1 }}` which on narrow viewports can cause text clipping
       - Add `flexShrink: 1` to the card style so it can shrink below minWidth on very narrow screens
       - Ensure the rationale Typography has no `numberOfLines` prop or `overflow: hidden` that would clip text
       - If the rationale text is clipped horizontally, ensure the parent View has `flexWrap` or the text component has proper flex behavior

    3. Update `tests/screens/the-showcase.test.tsx` if needed to ensure accordion content is still testable. The test should verify that requirement items are rendered (they already should be since the test renders the component).
  </action>

    - The Showcase test still passes
  </done>
