  <action>
    **Modal.tsx** — Fullscreen overlay using RN Modal:
    - Props: `visible: boolean`, `onClose: () => void`, `children: ReactNode`, `animationType?: 'fade' | 'slide'` (default 'fade')
    - RN `Modal` with `transparent={true}`, `visible`, `animationType`, `onRequestClose={onClose}`
    - Backdrop: `Pressable` filling screen with `bg-black/60` (semi-transparent black), `onPress={onClose}`
    - Content container: centered `View` with `flex-1 items-center justify-center px-4`
    - Children rendered inside a `Pressable` with `onPress={(e) => e.stopPropagation()}` to prevent backdrop close when tapping content (or use a nested View)
    - Test early: if RN Modal has z-index issues with Expo Router Stack (see pitfall 3 in research), use fallback approach with absolute-positioned View at root level

    **ModalContent.tsx** — Styled content wrapper for inside Modal:
    - Props: `title?: string`, `children: ReactNode`, `footer?: ReactNode`, `className?: string`, `onClose?: () => void`
    - Wrapper: `View className="bg-[#0a0a0a] border border-white/15 rounded-2xl w-full max-w-md overflow-hidden"`
    - Header (if title or onClose): `View className="flex-row items-center justify-between px-5 pt-5 pb-3"` with `Typography variant="h3"` for title and close icon button (MaterialIcons `close`, 20px)
    - Body: `View className="px-5 py-3"` wrapping children
    - Footer (if provided): `View className="px-5 pb-5 pt-2 border-t border-white/10"` wrapping footer
    - Width constrained by `max-w-md` (good for web, fills on mobile)

    **Alert.tsx** — Toast/notification component with semantic styling:
    - Props: `type?: 'success' | 'info' | 'warning' | 'error'`, `title?: string`, `message: string`, `visible?: boolean`, `onDismiss?: () => void`, `className?: string`
    - Semantic color map (same pattern as Badge but for alert backgrounds):
      ```
      success: { bg: 'bg-green-500/10', border: 'border-green-500/30', icon: 'check-circle', iconColor: '#4ade80' }
      info:    { bg: 'bg-blue-500/10',  border: 'border-blue-500/30',  icon: 'info',         iconColor: '#60a5fa' }
      warning: { bg: 'bg-yellow-500/10', border: 'border-yellow-500/30', icon: 'warning',    iconColor: '#facc15' }
      error:   { bg: 'bg-red-500/10',   border: 'border-red-500/30',   icon: 'error',        iconColor: '#f87171' }
      ```
    - Layout: `View` with `flex-row items-start px-4 py-3 rounded-xl border` + semantic classes
    - Left: MaterialIcons icon (20px) with semantic color
    - Center: `View className="flex-1 ml-3"` with title (Typography body, bold/text-white) and message (Typography caption, semantic text color)
    - Right (if onDismiss): close icon button (MaterialIcons `close`, 16px, `text-white/50`)
    - Animation: if `visible` is provided, wrap in `Animated.View` with Reanimated `FadeIn.duration(200)` entering and `FadeOut.duration(200)` exiting using `entering`/`exiting` layout animation props. If `visible` not provided, always render (static usage).

    **Dropdown.tsx** — Popover menu anchored to trigger:
    - Props: `trigger: ReactNode`, `items: { key: string; label: string; icon?: string; onPress: () => void; destructive?: boolean }[]`, `className?: string`
    - Uses RN `Modal` with `transparent={true}` and `animationType="fade"`
    - Trigger measurement pattern (from logbook Menu.tsx):
      1. `useRef` on trigger wrapper View
      2. On press, measure trigger position via `getBoundingClientRect()` on web (cast ref to `any`)
      3. Store `{ x, y, width, height }` in state
      4. Set `open = true`
    - Popover rendered inside Modal as:
      - Full-screen transparent `Pressable` backdrop (onPress closes)
      - `View` positioned absolutely: `top: triggerY + triggerHeight + 4`, `left: triggerX`, `minWidth: triggerWidth` (min 160px)
      - Styling: `bg-[#0a0a0a] border border-white/15 rounded-xl overflow-hidden`
    - Menu items: `Pressable className="flex-row items-center px-4 py-3 hover:bg-white/5"` with MaterialIcons icon (if provided) + `Typography variant="body"`. Destructive items use `text-red-400` color.
    - Item separator: `View className="h-px bg-white/10"` between items
    - Close on item press: call `item.onPress()` then `setOpen(false)`
    - **Popover position edge case:** If popover would extend below viewport, consider clamping. For now, simple below-trigger positioning is sufficient for the playground demo.

    **Update barrel exports:**
    - `feedback/index.ts`: Add Modal, ModalContent, Alert (keep existing SkeletonLine, SkeletonCard)
    - `layout/index.ts`: Add Dropdown (keep existing List, Tabs, Accordion, AccordionItem)
