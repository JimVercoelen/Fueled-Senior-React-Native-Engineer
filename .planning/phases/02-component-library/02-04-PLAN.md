    **Field.tsx** -- Shared wrapper for all form fields (from logbook Field.tsx pattern):
    - Props: `label?: string | ReactNode`, `helperText?: ReactNode`, `error?: { message?: string } | null`, `disabled?: boolean`, `className?: string`, `children: ReactNode`
    - Layout:
      - Label (if provided): `Typography variant="body" className="mb-2"` (or `variant="label"` for uppercase style -- use "body" to match logbook)
      - Children (the actual input)
      - Helper text (if provided AND no error): `Typography variant="caption" className="text-neutral-400 mt-1"`
      - Error (if error with message): `Typography variant="caption" className="text-red-400 mt-1"` showing `error.message`
    - Disabled: `opacity-50` on wrapper

    **TextField.tsx** -- Text input with Controller integration:
    - Generic: `<T extends FieldValues>` for type safety
    - Props: `control: Control<T>`, `name: FieldPath<T>`, `label?: string`, `helperText?: string`, `placeholder?: string`, `disabled?: boolean`, `secureTextEntry?: boolean`, `multiline?: boolean`, `numberOfLines?: number`, `className?: string`
    - Uses `Controller` from react-hook-form wrapping a RN `TextInput`
    - TextInput styling: `bg-white/5 border border-white/15 rounded-xl px-4 py-3 text-white font-body text-base`
    - Error state: border changes to `border-red-500` (use clsx conditional, NOT `!important` prefix -- see pitfall 7)
    - Focus state: border changes to `border-primary` (use `onFocus`/`onBlur` useState)
    - Disabled: `opacity-50` on TextInput (via Field wrapper)
    - `placeholderTextColor="rgb(156, 163, 175)"` (gray-400)
    - Pass `fieldState.error` to `Field` wrapper

    **Select.tsx** -- Single-select dropdown with popover:
    - Generic: `<T extends FieldValues>`
    - Props: `control: Control<T>`, `name: FieldPath<T>`, `label?: string`, `helperText?: string`, `placeholder?: string`, `options: { value: string; label: string }[]`, `disabled?: boolean`, `className?: string`
    - Uses `Controller` wrapping a trigger + popover
    - Trigger: `Pressable` styled like TextField (`bg-white/5 border border-white/15 rounded-xl px-4 py-3`), shows selected option label or placeholder in gray-400. Right side: MaterialIcons `expand-more` (20px, white/50).
    - Popover: SAME pattern as Dropdown.tsx -- `useRef` on trigger, `getBoundingClientRect()` on press, RN `Modal` transparent, absolute positioned popover below trigger
    - Popover content: `ScrollView` (maxHeight 240) with option items
    - Each option: `Pressable className="px-4 py-3 hover:bg-white/5"` with `Typography variant="body"`. Selected option has `bg-white/5` and a checkmark icon (MaterialIcons `check`, primary color).
    - On select: call `field.onChange(value)`, close popover
    - Error state: trigger border becomes `border-red-500`
    - Popover styling: `bg-[#0a0a0a] border border-white/15 rounded-xl overflow-hidden`

    **MultiSelect.tsx** -- Multi-select with chips and popover checkboxes:
    - Generic: `<T extends FieldValues>`
    - Props: `control: Control<T>`, `name: FieldPath<T>`, `label?: string`, `helperText?: string`, `placeholder?: string`, `options: { value: string; label: string }[]`, `disabled?: boolean`, `className?: string`
    - Uses `Controller` managing `string[]` value
    - Trigger: `Pressable` styled like TextField. If values selected, show as chips/tags (small `View` with `bg-white/10 rounded-lg px-2 py-1` + `Typography caption` + `x` close icon per chip, wrapped in flex-row flex-wrap). If empty, show placeholder.
    - Chip remove: remove value from array, call `field.onChange` with updated array
    - Popover: same pattern as Select. Content is a list of options with checkboxes.
    - Each option row: `Pressable className="flex-row items-center px-4 py-3"` with a custom checkbox indicator (square `View` 18x18 with `border border-white/30 rounded`, filled `bg-primary rounded` when checked) + `Typography variant="body"` for label.
    - Toggle: if value in array, remove it; if not, add it. Update via `field.onChange`.
    - "Done" button at bottom: `Button variant="primary" size="sm" label="Done"` that closes popover.
    - Popover footer: `View className="px-4 py-3 border-t border-white/10"` with Done button.

    **Toggle.tsx** -- Switch component with Controller:
    - Generic: `<T extends FieldValues>`
    - Props: `control: Control<T>`, `name: FieldPath<T>`, `label?: string`, `helperText?: string`, `disabled?: boolean`, `className?: string`
    - Uses `Controller` wrapping a toggle switch
    - Toggle track: `Pressable` -- `View` 48x28 with `rounded-full border` + conditional bg: off = `bg-white/10 border-white/20`, on = `bg-primary border-primary`
    - Toggle thumb: `Animated.View` 20x20 with `bg-white rounded-full`, animated translateX from 4px (off) to 24px (on) via Reanimated `useSharedValue` + `withTiming(duration: 200)`
    - Layout: `View className="flex-row items-center justify-between"` with `Field` wrapping label on left, toggle on right
    - On press: call `field.onChange(!field.value)`

    **Checkbox.tsx** -- Checkbox with Controller:
    - Generic: `<T extends FieldValues>`
    - Props: `control: Control<T>`, `name: FieldPath<T>`, `label?: string`, `helperText?: string`, `disabled?: boolean`, `className?: string`
    - Uses `Controller` wrapping a checkbox
    - Checkbox indicator: `View` 20x20 with `rounded border` -- unchecked: `border-white/30 bg-transparent`, checked: `bg-primary border-primary` with MaterialIcons `check` (14px, white) centered inside
    - Layout: `Pressable className="flex-row items-center gap-3"` -- checkbox indicator + `Typography variant="body"` for label
    - Wrap in Field for error display below
    - On press: `field.onChange(!field.value)`

    **forms/index.ts barrel:**
    ```typescript
    export { default as Field } from './Field';
    export { default as TextField } from './TextField';
    export { default as Select } from './Select';
    export { default as MultiSelect } from './MultiSelect';
    export { default as Toggle } from './Toggle';
    export { default as Checkbox } from './Checkbox';
    ```

    **IMPORTANT:** All form components use relative imports. Import Typography from `../ui/Typography`, Button from `../ui/Button`, Field from `./Field`. NO `@/` aliases.

    **IMPORTANT:** For Select and MultiSelect popover, reuse the exact same measurement pattern used in Dropdown.tsx (Plan 03). Do NOT create a separate Popover utility -- keep the pattern inline for simplicity. If the pattern is large, extract a shared hook (e.g., `usePopoverPosition`) in a utils file within forms/.

    **IMPORTANT:** Use `clsx` conditional for error borders (`error ? 'border-red-500' : 'border-white/15'`), NOT NativeWind `!` prefix. See pitfall 7.
