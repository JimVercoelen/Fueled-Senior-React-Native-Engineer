1. **Create `.github/workflows/ci.yml`**:
   Single workflow file with 3 jobs: lint, test, build-and-deploy.

   ```yaml
   name: CI/CD

   on:
     push:
       branches: [main]
     pull_request:
       branches: [main]

   jobs:
     lint:
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: apps/mobile
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: 'npm'
             cache-dependency-path: apps/mobile/package-lock.json
         - run: npm install --legacy-peer-deps
         - run: npm run lint

     test:
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: apps/mobile
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: 'npm'
             cache-dependency-path: apps/mobile/package-lock.json
         - run: npm install --legacy-peer-deps
         - run: npm test -- --forceExit

     build-and-deploy:
       needs: [lint, test]
       if: github.ref == 'refs/heads/main' && github.event_name == 'push'
       runs-on: ubuntu-latest
       defaults:
         run:
           working-directory: apps/mobile
       env:
         VERCEL_ORG_ID: ${{ secrets.VERCEL_ORG_ID }}
         VERCEL_PROJECT_ID: ${{ secrets.VERCEL_PROJECT_ID }}
       steps:
         - uses: actions/checkout@v4
         - uses: actions/setup-node@v4
           with:
             node-version: '20'
             cache: 'npm'
             cache-dependency-path: apps/mobile/package-lock.json
         - run: npm install --legacy-peer-deps
         - run: npm install -g vercel@latest
         - run: vercel pull --yes --environment=production --token=${{ secrets.VERCEL_TOKEN }}
         - run: vercel build --prod --token=${{ secrets.VERCEL_TOKEN }}
         - run: vercel deploy --prebuilt --prod --token=${{ secrets.VERCEL_TOKEN }}
   ```

   Key details:
   - `--legacy-peer-deps` in all npm install steps (React 19 peer conflict)
   - `--forceExit` on jest to prevent hanging workers
   - `build-and-deploy` only runs on push to main (not PRs)
   - `needs: [lint, test]` ensures tests must pass before deploy
   - Vercel secrets must be configured by user in GitHub repo settings
   - `cache-dependency-path` points to apps/mobile/package-lock.json for correct caching

2. **Create `apps/mobile/vercel.json`**:
   ```json
   {
     "buildCommand": "npx expo export -p web",
     "outputDirectory": "dist",
     "devCommand": "npx expo start --web",
     "cleanUrls": true,
     "framework": null,
     "rewrites": [
       {
         "source": "/:path*",
         "destination": "/"
       }
     ]
   }
   ```

   - `framework: null` prevents Vercel from auto-detecting and applying wrong build settings
   - `rewrites` handles SPA client-side routing (prevents 404 on direct URL access)
   - `output: "single"` in app.json already configures Expo for SPA output
   - `outputDirectory: "dist"` matches Expo's default export directory
  </action>
